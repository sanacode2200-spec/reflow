# リハビリ業務ドメイン知識

## 基本単位

- **1単位 = 20分**
- 患者1日上限：6単位（Phase1固定。9単位はPhase2以降）
- 療法士：1日18単位標準・1日24単位上限・週108単位上限
- 上限値はすべて `staffs.max_units_per_day` / `max_units_per_week` からDBで取得。ハードコード禁止。

## 施設基準区分

- Phase1は **運動器リハビリテーション料(II)** 固定
- `rehab_type` / `facility_grade` は ENUM で設計し Phase2 以降で拡張

## 疾患別区分と標準的算定日数

| 区分           | 標準算定日数 | 超過後の月上限 |
|----------------|-------------|----------------|
| 脳血管疾患等   | 180日       | 月13単位       |
| 運動器         | 150日       | 月13単位       |
| 廃用症候群     | 120日       | 月13単位       |
| 心大血管       | 150日       | 月13単位       |
| 呼吸器         | 90日        | 月13単位       |

- 残り14日前からアラート表示
- 超過後は月13単位上限を自動チェック

## 算定加算（算定漏れリスクが高いもの）

### 早期リハビリテーション加算
- **起算日から30日以内**が対象
- 患者登録時・記録入力時に自動判定してアラート表示

### 初期加算
- **起算日から14日以内**が対象
- 患者登録時・記録入力時に自動判定してアラート表示

### リハビリテーション総合計画評価料
- 月1回算定可能

### 目標設定等支援・管理料
- **要介護被保険者**が対象
- 標準算定日数の1/3経過後、未算定だと所定点数の90/100に減算

## 起算日

- 発症日 / 手術日 / 急性増悪日のいずれかを患者登録時に取得
- 早期加算・初期加算の判定起点
- 患者登録フォームで入力必須

## 離床減算

- 廃用症候群リハ等で離床が困難な場合は減算対象
- 実施記録ごとに `is_ambulatory`（boolean）で管理
- `false` = 減算フラグ。毎回の実施記録で入力

## 要介護被保険者

- 要介護フラグを患者情報で管理する
- 標準算定日数1/3経過後に目標設定等支援・管理料の算定状況をチェック

## 計画書様式

- 別紙様式21 / 別紙様式21の6 / 別紙様式23
- リハ開始後7日以内（遅くとも14日以内）に作成
- 3か月に1回以上更新・交付・診療録添付

> **⚠️ 令和8年度（2026年6月）診療報酬改定**：別紙様式23が様式21に統合予定。  
> 書類スキーマは JSONB で柔軟に設計し、改定時に差し替え可能にすること。

## スケジュール仕様

- **重複チェック**：時間の重複のみ（同療法士の予約間隔チェックは廃止）
- 例：10:01終了 → 10:05開始 でも ✅ OK
- 全曜日（月〜土）で予約作成・編集可能

## ステータス遷移

| ステータス | 遷移条件 |
|------------|----------|
| scheduled  | 予約作成時の初期状態 |
| draft      | いつでも一時保存可 |
| completed  | 単位数 + 記録本文が必須入力済みのときのみ |

## 実施記録

- SOAP形式 + テンプレート・前回コピー
- 開始・終了時刻は1分単位で手入力調整して保存（例：11:00〜11:21）
- 実施済み（completed）後の修正は `audit_logs` に JSONB で diff 保存
- 管理者のみ `audit_logs` 閲覧可能。療法士には「修正済み」バッジのみ表示

## Gotchas

- **みなし看護師制度**：制度開始が報告されている。算定要件への影響を確認・反映する必要あり（要調査）
- **9単位対応**は Phase2 以降。Phase1 は6単位固定
